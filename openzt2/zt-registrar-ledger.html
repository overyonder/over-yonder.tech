This page is the runtime type-system view of the register_type slice of the Binary Perimeter. Each row records a retail-game call that interns a native type name, links it to a parent type getter, and caches the resulting handle. The registry is used where shipped authored data crosses into native code: BFXML/XML tags, UI widgets, components, binders, modes, events, value types, and safe-handle surfaces can be looked up by registered name and hydrated through the original object/factory contract. After hydration, behavior continues through native C++ vtables, managers, and methods.
