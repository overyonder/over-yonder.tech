Binary Perimeter

The stripped Zoo Tycoon 2 binary's function, global/static, type-cache, vtable, and byte-map recovery surfaces. Source: zt-function-perimeter.tsv, zt-global-perimeter.tsv, zt-vtable-methods-full.tsv, zt-rtti-classes.tsv, and zt-byte-range-map.tsv.

Authority Model

The byte map is the address-fact archive. A function has an accepted exact identity only when a runtime-parent row in docs/analysis/zt-byte-range-map.tsv starts at that function entry. Candidate rows are proposed exact identities awaiting review. Region, provenance, vtable, and domain labels are triage context; they do not mean the function's concrete behavior has been recovered.

The class/type identity surface combines recovered names and vtable rows. A class/type can be identified without contributing vtable slots; BFSafeHandle<T> specializations are identified C++ template specializations with zero virtual slots. Slot-level bars count only rows that actually contain vtable slots.

Recovery Domains

Each bar below answers one completion question with one denominator. Solid bars use a closed denominator for that question. Striped bars use the currently known perimeter; future discoveries can expand the total. The byte-map bar sums the byte length of non-overlapping leaf partitions classified by the most specific covering map row; it is not a count of map rows or partitions.

Reading The Work Queue

Native function behavior tracks accepted and candidate exact recovery over known original .text function entries in the deobfuscated executable. The current tables do not keep a suspected-only function-entry bucket. Function entries and class methods overlap: the vtable column records virtual-method membership when a function body appears in an enumerated vtable slot. Absence from that column does not prove the function is not a constructor, static member, non-virtual member, or helper associated with a type. Rows marked wrapper-indirected are depepper support records connected to the wrapper indirection bar. Global/static references is a separate address surface from native function entries; its named type-cache-handle segment is refined by the type-cache handles bar below. Class/type recovery partitions the known class/type rows by recovered name and virtual-member identity in one denominator.

Globals / Statics

The global/static table is generated from absolute references in .text. Type-cache and identified global names come from byte-map rows; unidentified cells remain raw referenced addresses until a byte-map fact names them.

Per-Class Method Perimeter

Vtable method rows are generated from enumerated .rdata pointer runs and labelled from byte-map class evidence, constructor/vptr-store evidence, and the function perimeter. They are a method-shape view, not a second accepted function-name archive.
